# Development

## Local Checks

- Run tests: `make test`
- Byte-compile all package files: `make compile`

Both commands assume `emacs` is on `PATH`. They run in `-Q --batch` mode so the
results do not depend on your interactive init file.

## Repository Conventions

- Keep user-facing commands behind `M-x` or the `C-c l` prefix.
- Do not add bare single-key bindings to `blame-reveal-mode`.
- Keep optional modules optional. `blame-reveal-focus`, `-recursive`, and
  `-transient` must still be loadable independently.
- Use `defvar-local` for per-buffer state and keep shared state explicit.
- Keep byte-compilation warning-free.

## Testing Priorities

Add or update tests whenever changes touch:

- keymap behavior
- global mode enable/disable semantics
- transient and header-update interactions
- recursive blame state transitions
- focus mode navigation and exit behavior

The current regression tests live in `test/blame-reveal-test.el`.

## Manual Verification

For user-visible changes, test at least:

1. `M-x blame-reveal-mode` in a tracked file.
2. `C-c l f`, `C-c l n`, and `C-c l N` if focus mode is loaded.
3. `C-c l m` if transient support is loaded.
4. `M-x blame-reveal-global-mode` across multiple file buffers.
5. A large tracked file to confirm scrolling remains responsive.

## Artifact Hygiene

- Compiled `.elc` files are ignored by Git.
- Do not commit generated bytecode.
